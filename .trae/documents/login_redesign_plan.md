# 登录页重新设计计划

## 仓库研究结论

- 项目使用 **Next.js 14 + React 18 + TypeScript**
- UI 框架：**Ant Design 5**
- 样式工具：**Tailwind CSS**
- 现有登录页位于：`frontend-react/pages/login.tsx`
- 主题配置：`frontend-react/styles/theme.css`
- 当前登录页使用简洁的卡片式设计，包含账号和密码两个输入框

## 文件和模块修改

1. **主要修改文件**：
   - `frontend-react/pages/login.tsx` - 重新设计登录页 UI
   - `frontend-react/styles/theme.css` - 添加蓝色主题色变量

## 修改步骤

### 1. 更新主题配置
在 `theme.css` 中添加蓝色主题色变量：
- 主色调：蓝色系（如 `#1677ff`、`#0d6efd`）
- 背景渐变色
- 按钮和交互元素颜色

### 2. 重新设计登录页布局
- 采用现代 AI 平台风格，使用左右分栏布局
- 左侧：品牌展示区域（包含 Logo、标语、背景装饰）
- 右侧：登录表单区域
- 添加渐变背景和视觉效果

### 3. 优化表单设计
- 保留原有的表单逻辑和功能
- 改进输入框样式，增加聚焦效果
- 优化按钮设计，使用蓝色主题
- 添加加载状态和错误提示的视觉效果

### 4. 添加响应式支持
- 确保在移动设备上也有良好的显示效果
- 移动端切换为垂直布局

## 潜在依赖和考虑事项

- **Ant Design 主题配置**：需要确保 Ant Design 组件的主题色与自定义蓝色主题一致
- **现有功能保留**：登录逻辑、路由跳转、错误处理等功能必须完整保留
- **Dark Mode**：保持对深色模式的支持
- **Logo 使用**：项目已有 Logo 文件（`logo-horizontal.svg`、`logo-mark.svg`），可直接使用

## 风险处理

- 如果 Ant Design 主题覆盖有问题，可使用 CSS 自定义属性和 Tailwind 类名直接控制样式
- 确保所有表单验证和交互逻辑正常工作
- 测试在不同屏幕尺寸下的显示效果
